Setup Methods

Method 1: Direct Node.js Connection (Recommended)

Connect Cursor directly to the LocalMCP Node.js server for the fastest performance.

Step 1: Locate Your LocalMCP Installation

Find the path to your LocalMCP installation. The server file should be at:

# If you cloned the repo: /path/to/LocalMCP/dist/server.js # If you installed via npm: node_modules/localmcp/dist/server.js

Step 2: Configure Cursor MCP Settings

Open Cursor settings (Cmd/Ctrl + ,) and navigate to "Features" ‚Üí "Model Context Protocol". Add this configuration:

{ "mcpServers": { "localmcp": { "command": "node", "args": ["/absolute/path/to/LocalMCP/dist/server.js"], "env": { "NODE_ENV": "production", "LOCALMCP_LOG_LEVEL": "info" } } } }

Note: Replace /absolute/path/to/LocalMCP with your actual LocalMCP installation path.

Method 2: Docker Container Connection

Use Docker if you prefer containerized deployment or have complex environment requirements.

Step 1: Run LocalMCP in Docker

# Build the LocalMCP image docker build -t localmcp . # Run the container docker run -d --name localmcp-container \ -p 3000:3000 \ -v $(pwd):/workspace \ localmcp

Step 2: Configure Cursor for Docker

{ "mcpServers": { "localmcp": { "command": "docker", "args": [ "exec", "-i", "localmcp-container", "node", "/app/dist/server.js" ] } } }

Method 3: Development Mode

For developers working on LocalMCP itself or wanting the latest features.

Step 1: Run in Development Mode

# In your LocalMCP directory npm run dev

Step 2: Configure Cursor for Development

{ "mcpServers": { "localmcp": { "command": "node", "args": ["/path/to/LocalMCP/src/server.ts"], "env": { "NODE_ENV": "development", "TS_NODE_PROJECT": "/path/to/LocalMCP/tsconfig.json" } } } }

Advanced Configuration

Environment Variables

Customize LocalMCP behavior with environment variables:

{ "mcpServers": { "localmcp": { "command": "node", "args": ["/path/to/LocalMCP/dist/server.js"], "env": { "NODE_ENV": "production", "LOCALMCP_LOG_LEVEL": "info", "CONTEXT7_API_KEY": "your-context7-key", "CACHE_ENABLED": "true", "VECTOR_DB_URL": "http://localhost:6333" } } } }

Multiple Projects

Configure LocalMCP for different projects with different settings:

{ "mcpServers": { "localmcp-react": { "command": "node", "args": ["/path/to/LocalMCP/dist/server.js"], "env": { "PROJECT_TYPE": "react", "FRAMEWORK": "react" } }, "localmcp-vue": { "command": "node", "args": ["/path/to/LocalMCP/dist/server.js"], "env": { "PROJECT_TYPE": "vue", "FRAMEWORK": "vue" } } } }

Troubleshooting

Common Issues

Tools Not Appearing
  • Verify the path to server.js is correct
  • Check that LocalMCP is built: npm run build
  • Restart Cursor completely
  • Check Cursor's developer console for errors
Connection Errors
  • Ensure LocalMCP server is running
  • Check file permissions on the server file
  • Verify Node.js version compatibility
  • Check environment variables are set correctly
Performance Issues
  • Enable Context7 caching for faster responses
  • Use direct Node.js connection instead of Docker
  • Check available system resources
  • Monitor LocalMCP logs for bottlenecks

Debug Mode

Enable debug logging to troubleshoot issues:

{ "mcpServers": { "localmcp": { "command": "node", "args": ["/path/to/LocalMCP/dist/server.js"], "env": { "LOCALMCP_LOG_LEVEL": "debug", "DEBUG": "localmcp:*" } } } }

Best Practices

  • Keep LocalMCP Updated: Regularly update to get the latest features and improvements
  • Use Project-Specific Configs: Configure different settings for different project types
  • Monitor Performance: Use the admin console to monitor tool usage and performance
  • Learn from Patterns: Use the learn tool to capture successful coding patterns
  • Cache Documentation: Enable Context7 integration for faster framework documentation access
