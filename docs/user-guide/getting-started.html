Getting Started

This guide will help you get LocalMCP up and running quickly.

Prerequisites

  • Docker - For running the LocalMCP container
  • Cursor IDE - For AI-powered coding assistance
  • Node.js project - Your coding project (TypeScript, JavaScript, etc.)

Installation

Step 1: Pull the LocalMCP Image

docker pull localmcp:latest

Step 2: Run LocalMCP

docker run -p 3000:3000 -v $(pwd):/workspace localmcp

Step 3: Verify Installation

curl http://localhost:3000/health

Configuration

LocalMCP uses a simple configuration system. Create a localmcp.config.json file in your project root:

{ "context7": { "apiKey": "your-context7-api-key", "cacheEnabled": true }, "vectorDb": { "type": "qdrant", "url": "http://localhost:6333" }, "admin": { "enabled": true, "port": 3001 } }

Cursor IDE Setup

To use LocalMCP with Cursor IDE, you need to configure Cursor to connect to your LocalMCP server.

Step 1: Configure Cursor MCP Settings

Open Cursor settings and navigate to the MCP (Model Context Protocol) section. Add LocalMCP as a server:

{ "mcpServers": { "localmcp": { "command": "node", "args": ["/path/to/LocalMCP/dist/server.js"], "env": { "NODE_ENV": "production" } } } }

Step 2: Alternative - Use Docker

If you prefer to run LocalMCP in Docker, configure Cursor to connect to the container:

{ "mcpServers": { "localmcp": { "command": "docker", "args": ["exec", "-i", "localmcp-container", "node", "/app/dist/server.js"] } } }
