Pipeline Guide

Understanding the Dynamic Pipeline Engine that powers all LocalMCP tools.

What is the Dynamic Pipeline?

The Dynamic Pipeline Engine is the invisible brain of LocalMCP. It processes every request through a series of intelligent stages, each adding context and value to your request.

Pipeline Stages

📋 Retrieve.AgentsMD

Reads your project's AGENTS.md file to understand your coding guidelines and preferences.

🔍 Detect.RepoFacts

Analyzes your repository structure to understand the project type, framework, and dependencies.

📚 Retrieve.Context7

Fetches relevant documentation from Context7's cached library of frameworks and tools.

🧠 Retrieve.RAG

Searches your project's vector database for relevant code patterns, ADRs, and design decisions.

📖 Read.Snippet

Reads and analyzes the specific code files you're working with.

🤔 Reason.Plan

Creates an execution plan based on all the gathered context and your request.

✏️ Edit

Executes the planned changes to your code files.

✅ Validate

Validates the changes to ensure they meet quality standards and don't introduce errors.

🚪 Gate

Enforces policies and security checks before finalizing changes.

📝 Document

Generates documentation about the changes made.

🎓 Learn

Captures lessons learned and patterns for future use.

How It Works

  1. Request Received - You make a request through one of the 2 tools
  2. Context Gathering - Pipeline gathers context from multiple sources
  3. Analysis & Planning - AI analyzes the context and creates a plan
  4. Execution - Changes are made to your code
  5. Validation - Quality checks and policy enforcement
  6. Learning - Patterns are captured for future use

Benefits

  • Context-Aware - Every request understands your project and preferences
  • Quality Assured - Built-in validation and policy enforcement
  • Learning System - Gets smarter with every interaction
  • Transparent - You can see exactly what the pipeline is doing
